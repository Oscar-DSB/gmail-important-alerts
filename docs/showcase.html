Por qué existe

La arquitectura "correcta" te pide una tarjeta.
Esta no.

El diseño obvio para esto es Gmail → Pub/Sub → Cloud Function → Firestore: notificación push, en tiempo real, muy elegante. También exige una cuenta de facturación de Google Cloud activa, aunque el uso real nunca salga del tier gratuito.

Aquí se cambia el push por polling cada 30 minutos vía GitHub Actions, y Firestore por Upstash Redis. El resultado corre indefinidamente gratis, sin que nadie tenga que introducir un método de pago en ningún sitio.

El precio: hasta 30 minutos de retraso frente a una notificación instantánea. Para un asistente personal de "que no se me pase nada", es un cambio razonable.

Arquitectura

Cinco pasos, cero servidores

Sin watch() de Gmail que renovar: la sincronización incremental por historyId funciona igual sin ninguna suscripción push activa.

01 · CRON
Gmail history.list
GitHub Actions despierta cada 30 min y pide los mensajes nuevos desde el último historyId guardado.
02 · GRATIS
Filtros locales
Newsletters, publicidad y promos con unsubscribe se descartan sin gastar ni un token de Gemini.
03 · IA
Gemini clasifica
gemini-2.5-flash-lite puntúa 0-10, asigna categoría y urgencia. La respuesta se valida antes de confiar en ella.
04 · ESTADO
Upstash guarda
El historyId y cada mensaje procesado (TTL 30 días) evitan duplicados y reintentos innecesarios.
05 · AVISO
Telegram alerta
Solo si el score supera el umbral configurado. Todo lo demás queda en silencio.

Cómo decide

Nunca se confía ciegamente en el modelo

Gemini debe devolver JSON exacto. Se limpia de bloques Markdown, se valida campo a campo y cada valor fuera de rango se corrige antes de usarse:

// respuesta validada de importance_classifier.py
{
  "score": 9,
  "reason": "Confirmación de entrevista con fecha…",
  "category": "universidad",
  "requires_action": true,
  "urgency": "high"
}
Solo se envía alerta si score >= IMPORTANCE_THRESHOLD (por defecto 7). Si Gemini falla, el mensaje no se marca como procesado: se reintenta solo en el siguiente ciclo.
low informativo, sin acción necesaria
medium merece un vistazo esta semana
high entrevistas, fechas límite, universidad
critical seguridad, pagos, accesos comprometidos

Móntalo tú

Seis pasos, todos gratuitos

Credenciales OAuth de Gmail
Proyecto de Google Cloud sin facturación activada — Gmail API no la exige. Descarga credentials.json.
Genera el token
Ejecuta python scripts/setup_gmail_oauth.py, autoriza en el navegador.
Base de datos en Upstash
Redis gratis en console.upstash.com, copia REST URL y token.
API key de Gemini
Clave gratuita en aistudio.google.com/apikey.
Bot de Telegram
@BotFather → /newbot, y tu chat_id vía getUpdates.
Sube secretos y despliega
./scripts/setup_github_secrets.sh sube los seis secretos; el cron se activa solo al hacer push.
